# 输入步骤
1. 等待数据从网络中到达，当所等待的分组到达的时候，它就被复制到内核中的某个缓冲区中；
2. 把数据从内核的缓冲区拷贝到应用进程（用户）的缓冲区中。

# 阻塞式 I/O

![[阻塞式IO模型.png]]
- 进程阻塞于调用，等待调用返回

# 非阻塞式 I/O

![[非阻塞式IO模型.png]]
- 进程直接能得到调用的返回；
- 实际的结果需要多次调用，等待数据实际准备好并返回。
